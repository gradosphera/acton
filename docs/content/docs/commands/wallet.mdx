---
title: "Managing Wallets"
description: "Learn how to create and manage testnet wallets for your Acton projects"
---

The `wallet` command allows you to generate new wallets for testing purposes. It automatically creates a mnemonic file and updates your `Acton.toml` configuration.

<Callout type="warn">
  Wallets created by this command are configured for **testnet** by default. Coins in testnet have **NO VALUE**.
</Callout>

## Basic Usage

```bash
acton wallet new
```

This command will start an interactive prompt to guide you through the wallet creation process.

## Command Options

### New Wallet Options

| Option                | Type   | Description                                           |
|-----------------------|--------|-------------------------------------------------------|
| `--name <NAME>`       | String | Name of the wallet (default: prompts user)            |
| `--version <VERSION>` | String | Version of the wallet (default: prompts user)         |

**Supported wallet versions:**
- `v5r1` (Default)
- `v4r2`
- `v3r2`
- `highloadv2r2`
- And other historical versions (`v1r1`-`v3r1`, `highloadv1r1`-`highloadv2r1`)

## Examples

### Interactive Creation

Simply run the command without arguments to use the interactive mode:

```bash
acton wallet new
```

You will be prompted to:
1. Enter a name for your wallet
2. Select the wallet version from a list

### Non-interactive Creation

You can provide all necessary information via command-line flags to skip the interactive prompts:

```bash
# Create a v5r1 wallet named "deployer"
acton wallet new --name deployer --version v5r1

# Create a highload wallet
acton wallet new --name batch-sender --version highloadv2r2
```

## Output

When you create a new wallet, two things happen:

1.  **Mnemonic File**: A `{name}.mnemonic` file is created containing the 24-word secret phrase.
2.  **Configuration**: The wallet is added to your `Acton.toml` file.

```toml
[wallets.deployer]
kind = "v5r1"
workchain = 0
keys = { mnemonic-file = "deployer.mnemonic" }
```

## Getting Testnet Coins

Coins in testnet have no value, but you need them to pay for transaction fees (gas) when deploying contracts or sending messages.

To get testnet coins:

1.  Use the **[Testnet Giver Bot](https://t.me/testgiver_ton_bot)** on Telegram.
2.  Send your wallet address to the bot.
3.  The bot will send you some testnet TON.

For more methods (like web-based faucets), check the [official documentation](https://docs.ton.org/ecosystem/wallet-apps/get-coins#how-to-get-coins-on-testnet).

## Security

<Callout type="error">
  **SECURITY WARNING**

  The generated mnemonic is stored in **plain text** in the `.mnemonic` file.

  - **NEVER** commit mnemonic files to version control (git).
  - Acton automatically adds `*.mnemonic` to your `.gitignore` when initializing a project, but you should verify this.
  - These wallets are intended for **development and testing only**.
  - Do not use these wallets for mainnet funds or valuable assets.
</Callout>
