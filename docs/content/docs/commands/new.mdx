The `new` command initializes a new Tolk project. It sets up the directory structure, configuration files, and standard libraries required for development. You can choose from various templates to get started quickly.

## Basic Usage

```bash
acton new <PATH>
```

This command creates a new project in the specified directory. If the directory does not exist, it will be created.

## Command Options

| Option                 | Type   | Description                                                        |
|------------------------|--------|--------------------------------------------------------------------|
| `<PATH>`               | String | Directory to create the project in (use `.` for current directory) |
| `--name <NAME>`        | String | Project name (defaults to directory name)                          |
| `--description <DESC>` | String | Project description                                                |
| `--template <NAME>`    | String | Project template to use (empty, counter, jetton)                   |
| `--license <LICENSE>`  | String | Project license (MIT, Apache-2.0, etc.)                            |
| `--app`                | Bool   | Include the template's TypeScript app scaffold when available      |
| `--hooks`              | Bool   | Create `.githooks/` and install the default local Git hooks        |

## Interactive Mode

If you run `acton new` without providing all optional flags, it will launch an interactive wizard to guide you through the setup process.

```acton-cli
$ acton new my-project
? Project name: my-project
? Description: A TON blockchain project
? Template: counter
? License: MIT
? Include the TypeScript app scaffold? No
? Install the default Git hooks? No
```

If the selected template supports a TypeScript app scaffold and you do not pass
`--app`, Acton asks whether you want to include it. In non-interactive
environments, the app scaffold is not included unless you pass `--app`
explicitly.

If `git` is available, and you do not pass `--hooks`, Acton also asks whether it
should create and install the default project-local Git hooks. In
non-interactive environments, hooks are not installed unless you pass
`--hooks` explicitly.

## Examples

### Interactive creation

```bash
# Create project in a new directory 'my-token'
acton new my-token

# Initialize project in current directory
acton new .
```

### Non-interactive creation

You can skip prompts by providing all necessary flags:

```bash
# Create a minimal project
acton new my-project --template empty --license MIT

# Create a Jetton project with description
acton new token-project --template jetton --desc "My Awesome Token"

# Create the counter template with the TypeScript app scaffold
acton new counter-dapp --template counter --app

# Create a project and install the default Git hooks immediately
acton new my-project --template empty --license MIT --hooks
```

## Templates

Acton provides several built-in templates:

| Template  | Description                                                                                                                            |
|-----------|----------------------------------------------------------------------------------------------------------------------------------------|
| `empty`   | Minimal structure with a single empty contract. Best for experienced users.                                                            |
| `counter` | Simple counter contract with tests and deployment script. Good for learning. Supports an optional TypeScript app scaffold via `--app`. |
| `jetton`  | Complete Jetton (token) implementation with Minter and Wallet contracts.                                                               |

## Project Structure

### Contract-only layout

A standard project contains the following structure:

<Files>
  <Folder name=".acton" defaultOpen>
    <File name="tolk-stdlib/" />
    <File name="..." />
  </Folder>
  <Folder name="contracts" defaultOpen>
    <File name="contract.tolk" />
  </Folder>
  <Folder name="scripts" defaultOpen>
    <File name="deploy.tolk" />
  </Folder>
  <Folder name="tests" defaultOpen>
    <File name="contract.test.tolk" />
  </Folder>
  <File name="Acton.toml" />
  <File name="LICENSE" />
  <File name=".gitignore" />
</Files>

- **`.acton/`**: Standard libraries and internal files
- **`contracts/`**: Smart contract source files
- **`scripts/`**: Deployment and utility scripts
- **`tests/`**: Test files
- **`Acton.toml`**: Project configuration
- **`LICENSE`**: License file
- **`.gitignore`**: Git configuration

### Counter Template With `--app`

When you run `acton new <PATH> --template counter --app`, Acton generates a
split contract + frontend project:

<Files>
  <Folder name=".acton" defaultOpen>
    <File name="tolk-stdlib/" />
    <File name="..." />
  </Folder>
  <Folder name="contracts" defaultOpen>
    <Folder name="src" defaultOpen>
      <File name="counter.tolk" />
      <File name="types.tolk" />
    </Folder>
    <Folder name="tests" defaultOpen>
      <File name="counter.test.tolk" />
      <Folder name="wrappers">
        <File name="Counter.tolk" />
      </Folder>
    </Folder>
    <Folder name="scripts" defaultOpen>
      <File name="deploy.tolk" />
    </Folder>
  </Folder>
  <Folder name="app" defaultOpen>
    <File name="src/App.tsx" />
    <File name="..." />
  </Folder>
  <Folder name="wrappers" defaultOpen>
    <File name="Counter.ts" />
  </Folder>
  <File name="package.json" />
  <File name="package-lock.json" />
  <File name="Acton.toml" />
  <File name="LICENSE" />
  <File name=".gitignore" />
</Files>

- **`contracts/src/`**: Tolk contract sources and shared types
- **`contracts/tests/`**: Tolk tests and generated Tolk wrappers
- **`contracts/scripts/`**: Deployment and utility scripts for the contracts
- **`wrappers/`**: Generated TypeScript wrappers used by the app
- **`app/`**: React + Vite frontend
- **`package.json` / `package-lock.json`**: Frontend toolchain and pinned npm dependencies

After creation, install frontend dependencies with:

```bash
npm install
```

## Git Hooks

When you pass `--hooks` or confirm the interactive prompt, `acton new`:

- creates `.githooks/pre-commit`
- writes the default hook that runs `acton check` and `acton fmt --check`
- sets the repository-local `core.hooksPath` to `.githooks`

This requires the `git` command to be available in `PATH`.
