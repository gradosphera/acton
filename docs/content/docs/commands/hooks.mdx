The `hooks` command helps you manage project-local Git hooks through Git's
`core.hooksPath` setting. Acton expects hooks to live in `.githooks/` in the
resolved project root.

## Commands

### `acton hooks new`

Creates a `.githooks/` scaffold in the resolved project root.

```bash
acton hooks new
```

`acton hooks new` only works when the resolved project root already contains a
local `.git` directory.

If local Git hooks are already configured for the repository, `acton hooks new`
fails and asks you to run `acton hooks uninstall` first.

If `--template` is omitted, Acton opens an interactive selector with two
options:

- `empty` creates an empty `.githooks/pre-commit`
- `default` creates `.githooks/pre-commit` with a starter hook:

```sh
#!/bin/sh
set -e
acton check
acton fmt --check
```

#### Options

| Option                        | Type   | Description                      |
|-------------------------------|--------|----------------------------------|
| `--template <empty\|default>` | String | Hook scaffold template to create |

#### Examples

```bash
# Choose template interactively
acton hooks new

# Create an empty pre-commit hook
acton hooks new --template empty

# Create a default-made pre-commit hook
acton hooks new --template default
```

<Callout type="warn">
If `.githooks/` already exists, `acton hooks new` fails instead of overwriting
existing hooks. If `.githooks/pre-commit` already exists, delete that file
before running `acton hooks new`.
</Callout>

---

### `acton hooks install`

Configures the Git repository for the resolved project root to use `.githooks/`:

```bash
acton hooks install
```

`acton hooks install` only works when the resolved project root already contains
a local `.git` directory.

Internally this runs:

```bash
git config --local core.hooksPath .githooks
```

On success, Acton prints:

```text
Git hooks installed successfully from .githooks
```

If `core.hooksPath` is already set locally for this repository, `acton hooks install`
fails and asks you to run `acton hooks uninstall` first.

Equivalent local paths that resolve to the same hooks directory, such as
`./.githooks` or an absolute path to the same directory, also count as already
installed.

If `.githooks/` does not exist yet, `acton hooks install` fails and asks you to
run `acton hooks new` first.

---

### `acton hooks status`

Reports whether the Git repository for the resolved project root already points
`core.hooksPath` to `.githooks`:

```bash
acton hooks status
```

`acton hooks status` only works when the resolved project root already contains
a local `.git` directory.

Possible outputs:

```text
Git hooks are installed
```

or:

```text
Git hooks are not installed
```

`status` checks the repository's local Git config only. Global or system
`core.hooksPath` values do not count as installed for the current project.
Equivalent local paths that resolve to the same `.githooks/` directory count as
installed.

---

### `acton hooks uninstall`

Removes the `core.hooksPath` override from the Git repository for the resolved
project root:

```bash
acton hooks uninstall
```

`acton hooks uninstall` only works when the resolved project root already
contains a local `.git` directory.

Internally this runs:

```bash
git config --local --unset-all core.hooksPath
```

On success, Acton prints:

```text
Git hooks uninstalled successfully
```

If no local `core.hooksPath` is configured, `uninstall` succeeds and prints:

```text
Git hooks are not installed
```

## Recommended Workflow

```bash
acton hooks new --template default
acton hooks install
acton hooks status
```

After that, Git runs hooks from `.githooks/` for the current repository.
